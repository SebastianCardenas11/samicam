INSTRUCCIONES PARA LA MIGRACIÓN DE FUNCIONARIOS A DOS TABLAS SEPARADAS
=======================================================================

Este documento contiene las instrucciones para migrar la estructura actual de funcionarios
a un sistema con dos tablas separadas: una para funcionarios de planta y otra para funcionarios OPS.

PASOS A SEGUIR:
--------------

1. RESPALDO DE LA BASE DE DATOS
   - Antes de realizar cualquier cambio, haga un respaldo completo de su base de datos.
   - Puede usar phpMyAdmin para exportar toda la base de datos samicam.

2. EJECUTAR EL SCRIPT DE ACTUALIZACIÓN
   - Abra phpMyAdmin y seleccione la base de datos samicam
   - Vaya a la pestaña "SQL" y ejecute el contenido del archivo db_update.sql
   - Este script realizará las siguientes acciones:
     * Renombrar la tabla tbl_funcionarios a tbl_funcionarios_planta
     * Crear una nueva tabla tbl_funcionarios_ops
     * Migrar los funcionarios OPS a la nueva tabla
     * Eliminar los funcionarios OPS de la tabla de planta
     * Actualizar las tablas relacionadas (vacaciones, permisos, etc.)

3. VERIFICAR LA MIGRACIÓN
   - Después de ejecutar el script, verifique que:
     * La tabla tbl_funcionarios_planta contiene solo funcionarios de planta
     * La tabla tbl_funcionarios_ops contiene solo funcionarios OPS
     * Las relaciones entre tablas funcionan correctamente

4. ACTUALIZAR LOS ARCHIVOS DEL SISTEMA
   - Los siguientes archivos ya han sido actualizados para trabajar con la nueva estructura:
     * Models/FuncionariosPlantaModel.php
     * Models/FuncionariosOpsModel.php
     * Models/VacacionesModel.php
     * Models/FuncionariosPermisosModel.php
     * Models/FuncionariosOpsPermisosModel.php (nuevo)
     * Controllers/FuncionariosOpsPermisos.php (nuevo)

5. CREAR DIRECTORIOS NECESARIOS
   - Cree el siguiente directorio si no existe:
     * Views/FuncionariosOpsPermisos

6. PROBAR EL SISTEMA
   - Acceda al sistema y verifique que:
     * El módulo de Funcionarios Planta muestra solo funcionarios de planta
     * El módulo de Funcionarios OPS muestra solo funcionarios OPS
     * Las funcionalidades de vacaciones y permisos funcionan correctamente para funcionarios de planta
     * Los funcionarios OPS no tienen acceso a vacaciones ni permisos

NOTAS IMPORTANTES:
-----------------
- Los funcionarios OPS no tienen permisos, vacaciones, fecha de vacaciones ni periodos de vacaciones
- Si encuentra algún error, restaure el respaldo de la base de datos y contacte al soporte técnico
- Esta migración es irreversible una vez completada, por lo que es crucial hacer un respaldo antes de comenzar

Para cualquier consulta o problema, contacte al equipo de soporte técnico.
